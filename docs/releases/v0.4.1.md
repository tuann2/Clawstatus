# Clawstatus 0.4.1

Clawstatus 0.4.1 is a reliability update for Claude Code and Codex usage
polling.

## Highlights

- Adds a 30-second timeout and task cancellation for Claude Code usage calls,
  preventing a stalled CLI from stopping future refreshes.
- Detects an exited Codex app-server immediately instead of waiting for the
  full request timeout, and releases completed timeout work promptly.
- Labels restored snapshots as **Cached** until a provider refresh succeeds
  and formats snapshot age in seconds, minutes, or hours.
- Retries every 60 seconds normally, then backs off to at most 5 minutes when
  both providers repeatedly fail. **Refresh now** retries immediately and
  restores the normal interval.
- Reports unsupported provider output more clearly and logs only the provider
  error category, never usage reports, responses, credentials, or account
  details.
- Honors the timezone included in Claude Code reset timestamps.
- Reads the app version from the application bundle when initializing the
  Codex app-server client.

## Privacy and authentication

Clawstatus delegates authentication to the installed official CLIs. It never
reads, stores, logs, or refreshes OAuth tokens and never reads
`~/.codex/auth.json`. Only the last successful usage snapshots and UI
preferences are saved locally.

## Compatibility

- Apple Silicon only
- macOS 13 Ventura or newer
- Claude Code with headless `/usage` support, Codex CLI, or both

## Install with Homebrew

```bash
brew tap tuann2/clawstatus https://github.com/tuann2/Clawstatus
brew update
brew upgrade --cask clawstatus
xattr -dr com.apple.quarantine /Applications/Clawstatus.app
open -a Clawstatus
```

The build is ad-hoc signed and not Apple-notarized. The scoped `xattr` command
is therefore required after installation or upgrade; it does not disable
Gatekeeper system-wide.
