# Clawstatus 0.4.0

Clawstatus 0.4.0 adds Codex usage monitoring alongside Claude Code while
keeping the app native, small, and dependency-free.

## Highlights

- Shows Claude and Codex remaining usage together in the menu bar, for example
  `C 76% · X 47%`.
- Reads Codex rate limits through the official short-lived
  `codex app-server --stdio` interface.
- Displays every available provider window in the floating card, including
  dynamically named Codex primary and secondary windows.
- Keeps provider refreshes and cached snapshots independent, so one unavailable
  CLI does not hide the other provider.
- Safely terminates the Codex app-server after each refresh and guards writes
  during cancellation.
- Retains the compact card, opacity controls, and remembered UI preferences
  introduced in 0.3.0.

## Privacy and authentication

Clawstatus delegates authentication to the installed official CLIs. It never
reads, stores, logs, or refreshes OAuth tokens and never reads
`~/.codex/auth.json`. Only the last successful usage snapshots and UI
preferences are saved locally.

Install and sign in to Claude Code, Codex CLI, or both. Clawstatus displays
whichever provider snapshots are available.

## Compatibility

- Apple Silicon only
- macOS 13 Ventura or newer
- Claude Code with headless `/usage` support, Codex CLI, or both

## Install with Homebrew

```bash
brew tap tuann2/clawstatus https://github.com/tuann2/Clawstatus
brew install --cask clawstatus
xattr -dr com.apple.quarantine /Applications/Clawstatus.app
open -a Clawstatus
```

The build is ad-hoc signed and not Apple-notarized. The scoped `xattr` command
is therefore required after installation or upgrade; it does not disable
Gatekeeper system-wide.
