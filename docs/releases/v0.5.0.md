# Clawstatus 0.5.0

Clawstatus 0.5.0 adds provider controls, a widget-style HUD, and a shared CLI
process foundation for Claude Code and Codex.

## Highlights

- Adds a Settings popover and right-click **Providers** menu for independently
  enabling Claude Code and Codex usage.
- Stops launching a disabled provider CLI, preserves its cached snapshot, and
  restarts polling immediately when the provider is enabled again.
- Rebuilds the Full HUD as a dynamic card per provider with independent
  **Live**, **Cached**, and **Unavailable** states.
- Updates Compact mode to show one summary row per enabled provider and adds an
  empty state when every provider is disabled.
- Keeps the menu-bar label compact and replaces it with the app icon when all
  providers are disabled.
- Adds a normalized provider registry while preserving the existing Claude and
  Codex cache file formats.
- Consolidates timeout, cancellation, stdout/stderr draining, and exactly-once
  completion in a shared `CLIProcessRunner`.
- Supports current Claude Code `/usage` behavior, including a session with no
  reset timestamp, while preventing prompt/session transcript writes.
- Restores **Open Terminal** recovery for missing provider CLIs and prevents
  stale polling results from re-enabling or overwriting a disabled provider.

## Privacy and authentication

Clawstatus delegates authentication to the installed official CLIs. It never
reads, stores, logs, or refreshes OAuth credentials. Raw CLI output and stderr
are not logged. Only normalized usage snapshots and UI preferences are saved.

Claude Code runs with `--safe-mode` and
`CLAUDE_CODE_SKIP_PROMPT_HISTORY=1`; Codex usage continues through the official
short-lived `codex app-server --stdio` interface.

## Scope

- Antigravity remains disabled because its current CLI does not provide a
  deterministic, machine-readable quota command that can be polled without
  invoking a model.
- A real WidgetKit desktop extension is not included. Version 0.5.0 delivers
  the widget-style floating HUD without changing the existing ad-hoc signing
  and SwiftPM distribution model.

## Compatibility

- Apple Silicon only
- macOS 13 Ventura or newer
- Claude Code with headless `/usage` support, Codex CLI, or both

## Install or upgrade with Homebrew

```bash
brew tap tuann2/clawstatus https://github.com/tuann2/Clawstatus
brew update
brew upgrade --cask clawstatus
xattr -dr com.apple.quarantine /Applications/Clawstatus.app
open -a Clawstatus
```

The build is ad-hoc signed and not Apple-notarized. The scoped `xattr` command
is required after installation or upgrade and does not disable Gatekeeper
system-wide.
